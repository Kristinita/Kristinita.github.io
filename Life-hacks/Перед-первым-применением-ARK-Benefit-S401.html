1. Введение

1.1. Для чего предназначена данная инструкция

Стоковая прошивка телефона ARK Benefit S401 содержит вредоносное ПО — без Вашего согласия на телефон будут устанавливаться программы. Данная проблема возникла не только у меня, но и у многих прочих участников форума 4PDA, см., например, сообщения 1, 2, 3, 4, 5, 5, 6, 7.

Если не желаете мириться с постоянными загрузками нежелательного ПО, нужно что-то делать.

1.2. Для кого предназначена данная инструкция

В основном, для пользователей, не имеющих большого опыта перепрошивок. К сожалению, сообщения по ссылкам в шапке 4PDA могут быть не совсем понятными для неопытных пользователей.

Если же Вы опытный юзер и заметили ошибки/недочёты в данной статье, просьба в конструктивном ключе сообщить о них автору.

1.3. Необходимое оборудование

Автор статьи лично выполнял описанные ниже действия на следующем оборудовании, и у него получилось.

  1. ARK Benefit S401 со стоковой прошивкой — собственно, телефон, о котором идёт речь;
  2. Windows 10 LTSB Enterprise EN;

  3. Нужно как-нибудь перенести файл прошивки на SD-карту.

  4. Если телефон нормально работает, удобнее всего переносить файлы с компьютера на телефон и обратно с помощью программы FTP Server (Free).

  5. Если телефоном не получается пользоваться в обычном режиме, — после ошибки YouTube или неудачном сбросе настроек — я беру устройство, поддерживающее MicroSD (для меня это скрытая камера) → вставляю в него MicroSD из телефона → подключаю устройство через USB к компьютеру → переношу на MicroSD нужные файлы → отключаю устройство от компьютера → вставляю MicroSD обратно в телефон.
  6. Вероятно, переносить файлы без дополнительного (помимо USB-кабеля) оборудования можно посредством adb. Сам не пробовал, не могу сказать, какие подводные камни ожидают.

2. Чего не стоит делать

Не повторяйте моих ошибок.

2.1. Не удаляйте самостоятельно заражённые файлы, если не уверены в последствиях.

я пытался удалять вредоносное ПО следующим образом.

  1. Программой RootAppDelete удалил приложение YouTube,
  2. Удалял YouTube программой Lucky Patcher,
  3. Просканировал телефон антивирусом Dr.Web и удалил вручную, при помощи Root Explorer, файлы, на которые указал Dr.Web.

Во всех трёх случаях стала постоянно появляться ошибка приложения YouTube, которая не давала ничего делать на данном телефоне.

2.2. Делайте бэкапы перед тем, как совершать рискованные действия

Под «рискованными действиями» понимаются, в частности, перепрошивка, сброс до заводских настроек, монтирование разделов, на конкретно данной модели телефона — удаление вирусных приложений.

я делаю 2 бэкапа:

  1. При помощи oandbackup,
  2. В кастомном Recovery.

2.3. Не устанавливайте официальные прошивки

V007 и V006, которые рекомендует официальная группа ARK ВКонтакте. В них также содержится вредоносное ПО.

2.4. В прошивке VERTEX баги

Данные баги не воспроизводятся в прошивке CyanogenMod 12.1. Кроме того в VERTEX присутствуют GAPPS, отсутствующие в прошивке CyanogenMod 12.1.

3. Установка кастомного recovery и перепрошивка

3.1. Зачем это нужно

  1. Без кастомного recovery не получится установить новую прошивку. Что ещё позволяет TWRP recovery.
  2. Даже если будет выложен безопасный способ удаления вредоносного ПО в данной модели телефона, в официальных прошивках останутся GAPPS. мне без них лучше. В прошивке CyanogenMod 12.1 GAPPS отсутствуют.

3.2. Установка кастомного recovery

Приступаем к действиям. Если Вы успели сохранить на телефон что-то нужное

я установил TWRP 3.1.0. Проделываем действия по этой инструкции. Моменты, которые у меня вызвали затруднения:

3.2.1. Установка драйверов

я делал так. С выключенным телефоном заходил в Device Manager → подключил выключенный телефон по USB → в Device Manager на время около секунды появлялся пункт Ports (COM & LPT) → Preloader USB VCOM Port (COM6).

Успевал кликнуть на Preloader USB VCOM Port (COM6) → Driver → Update Driver... → Browse my computer for driver software → выбирал папку с драйверами, которую скачал в инструкции, → Поставил галочку на Include subfolders → Next → получал окно The best driver software for your device is already installed.

Несмотря на это сообщение, установить кастомное recovery получилось. Видимо, если драйвер уже установлен, можно приступать к дальнейшим действиям.

3.2.2. Кнопка Download

После пункта инструкции Снимаем все галки кроме recovery, и нажав на путь в той же строке, выбираем в открывшемся окне скачанный ранее образ кастомного рекавери. следует нажать на большую зелёную кнопку Download

После нажатия подключил по USB выключенный телефон → программа SP Flash Tool моментально определила телефон (видно по жёлтой полосе внизу окна программы) → загрузка успешно завершилась.

3.3. Установка CyanogenMod 12.1

Все прошивки из шапки не пробовал, возможно, есть те, которые работают лучше.

Инструкция, прошивка; GAPPS по ссылке устанавливать не стал. Дополнения к инструкции:

3.3.1. Приложения

После успешной установки данной прошивки все иконки на рабочем столе, которые у меня были раньше, удалились. Это может оказаться неприятной неожиданностью: когда я перепрошивался на Cosmic ROM в Samsung GT-I8262 Galaxy Core, данные на рабочем столе остались нетронутыми (точно не помню, возможно, из-за того, что не сделал все вайпы, ну и точно я не форматировал раздел system).

Чтобы восстановить все иконки, которые у меня были на рабочем столе, в oandbackup я сделал восстановление из бэкапа.

3.3.2. Вайпы

После пункта Качаем прошивку из шапки, из раздела "Неофициальные прошивки" и кладём на карту памяти, вероятно, лучше сделать вайпы data/Factory reset, Cache Partition, Dalvik Cache через кастомное recovery.

3.3.3. Форматирование раздела system

После вайпов необходимо через кастомное рекавери отформатировать раздел system, иначе после перепрошивки будет висеть логотип ARK, и дальше дело не двинется.

3.3.4. Окончание перепрошивки

В остальном инструкция для меня понятна. После успешной установки новой прошивки можно вновь установить Dr.Web и просканировать систему на наличие вирусов. На сей раз их быть не должно.

Если Вы делали бэкап, можно восстановить данные.
